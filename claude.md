# Portfolio Project — Claude Execution Guide

## Project Type

Static professional portfolio website deployed via GitHub Pages.

## Core Rules (Scope Lock)

This project MUST remain:

* Static only (NO backend, NO database, NO server logic)
* Fast and lightweight
* Clean and professional
* Minimalist dark theme
* SEO friendly
* Fully responsive
* Recruiter-focused structure

DO NOT introduce:

* Frameworks unless explicitly requested
* CMS
* Authentication
* API integrations
* Heavy animations
* Unnecessary complexity

---

## Tech Stack

Base stack:

* HTML5 semantic structure
* Modern CSS (variables, flex/grid, responsive)
* Minimal vanilla JavaScript (only if necessary)
* Static assets only

---

## Design Principles

* Dark, minimalist, professional
* High readability typography
* Clean spacing
* Clear visual hierarchy
* No clutter
* Mobile-first
* Fast loading
* Accessibility-aware

---

## Site Structure

Pages:

* Home
* About
* Projects
* Skills (optional but recommended)
* Contact

---

## Performance Targets

* Lighthouse Performance ≥ 90
* SEO ≥ 90
* Accessibility ≥ 90
* Best Practices ≥ 90

---

## SEO Basics Required

* Proper semantic HTML
* Meta title + description
* OpenGraph tags
* Structured data (Person)
* Sitemap
* Robots.txt

---

## Code Quality Rules

* Clean and readable
* No unused code
* No inline styles (unless justified)
* Reusable CSS variables
* Consistent spacing system
* Comment only when meaningful

---

## Deployment Target

GitHub Pages using repository:

`username.github.io`

The output MUST be static and deployable without build step unless explicitly instructed.
